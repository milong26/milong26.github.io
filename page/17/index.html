图中判断成环

求最小生成树的Kruskal算法里面遇到，这边写一下 无向图 DFS 对无向图有环=某个结点有一条边指向已访问过的结点，并且这个已访问过的结点不是上一步访问的结点 so问题可以转换成，如何区分已经访问过的节点，需要区分相邻的sibling、parent和间接的grandparent，当某节点有个sibling和某个grandparnt相同时，就是个环 修饰节点颜色 我们不能仅仅使用一个 bool 数组来表示结点是否访问过。规定每个结点都拥有三种状态，白、灰、黑。开始时所有结点都是白色，当访问过某个结点后，该结点变为灰色，当该结点的所有邻接点都访问完，该节点变为黑色。 那么我们的算法可以表示为：如果在遍历的过程中，发现某个结点有一条边指向灰色节点，并且这个灰色结点不是上一步访问的结点，那么存在环。 为什么：假设灰色节点为G，它为灰色表示至少已访问过其中一个child节点，其中一个表示为parent，现在遍历到一个节点为grandchild，它可以指向G就表示它的sibling=grandparent，同时G不是grandchild上一步所访问的节点，so出现了环，这样正着推ok 反过来推，为什么需要3个状态，如果单纯两个状态就是下面的暴力递归。 再梳理一遍就能发现对DFS来说，就好比祖母和同学是同一个人这样 没有修饰，直接暴力递归 节点就两个状态，访问过、没访问过。 我看不懂 个人理解： 更加纯正的DFS，不管三七二十一先遍历，遍历过了就给它标记成visited，然后一旦在遍历的时候遇到即将访问的point满足1. visited过了，2.不是直系parent，就直接输出是环，要等到所有遍历走过了才确认不是环。 BFS 和dfs相比主要的思路是一样的 BUR，BFS需要增加一个parent数组（不是DFS里面用单个parent记录刚访问过的节点） BFS相比DFS用得不怎么多，有空再钻研 Union-Find 也叫并查集。 对于无向图来说，在遍历边（u-v）时，如果结点 u 和结点 v 的parent相同，那么结点 u 和结点 v 在同一个环中。 图的全部顶点都各自属于一个集合；然后依次检查图中的所有边，如果边关联的两个顶点不属于同一个集合，就合并两个顶点所在的集合。若图中存在环，则一定会在检测到某条边的时候，发现它关联的两个顶点早已属于同一个集合。 并查集我不会啊 首先初始化所有元素的根为-1，-1代表根节点，接下来对于图中的每一条边(v1,v2)都并入集合，并入的方式为查找v1和v2的根节点，然后让v2的根节点作为v1的根节点，查找根节点的过程为：如果当前的结点根为-1，说明这个结点就是根，直接返回，否则再继续查找结点parent的根，直到找到root结点，这里因为只是判断环路，不需要压缩路径。 int findSet(int x){ if(Parent[x] == -1){ return x; 最小生成树

满足两点之间有且仅有一条简单路径的结构只有树 且这棵树包含 n 个节点。我们称这棵树为给定的图的生成树，其中总权值最小的生成树，我们称其为最小生成树。 最小生成树关键在避免环 生成树 定义 一个连通图的生成树是一个极小的连通子图，它包含图中全部的n个顶点，但只有构成一棵树的n-1条边。 性质 生成树的属性 一个连通图可以有多个生成树； 一个连通图的所有生成树都包含相同的顶点个数和边数； 生成树当中不存在环； 移除生成树中的任意一条边都会导致图的不连通， 生成树的边最少特性； 在生成树中添加一条边会构成环。 对于包含n个顶点的连通图，生成树包含n个顶点和n-1条边； 对于包含n个顶点的无向完全图最多包含$n^{n-2}$颗生成树。 证明$n^{n-2}$： 为什么是n^n-2），一篇里面塞不下:D 最小生成树 定义 所谓一个带权图的最小生成树，就是原图中边的权值最小的生成树 ，所谓最小是指边的权值之和小于或者等于其它生成树的边的权值之和。 求解 Kruskal 贪心找边，从小到大加入边 按边长度排序 从小到大，if加入e之后没有形成环，e加入G' 最后得到G‘ 问题是判断成环，见其它post Prim 加点，把点分成两坨A,B集合，A表示已经成了，B是待选的，要做的事从B里面挑一个点，满足它到A的点里面，有个边距离是最短的 （理解成长三角经济圈画好了，接下来先从周边里挑一个离长三角圈里面的城市最近的，这么想有点奇怪） 关键在怎么从待选里找距离最短的点 思路是用一个点个数大小的数组，没有直接连用∞，否则记录最小值 注意区分已访问过的点 这个为什么不会成环？把点分成两坨了，找的都是新点 示例与代码 测试数据 1 6 10 1 2 28 6 5 25 5 7 24 2 7 14 2 3 16 3 4 12 4 7 18 4 5 22 代码 使用Kruskal算法 #include<iostream> #include<algorithm> using namespace std; const int maxe = 100; int fa[maxe], n, m, ans = 0; struct edge { int u, v, w; }e[maxe]; bool cmp(edge a, edge b) { return a.w<b.w; } int get(int x)//用处是看它是否成环的，nb { if (fa[x] == x) return x;//x的根为它本身，x就是该集的根 return fa[x] = get(fa[x]);//路径压缩 } int main() { cin >> n >> m; for (int i = 1; i <= n; i++) fa[i] = i;//初始化 for (int i = 1; i <= m; i++) { cin >> e[i].u >> e[i].v >> e[i].w; } sort(e + 1, e + 1 + m, cmp);//升序排序 for (int i = 1; i <= m; i++) { int u = get(e[i].u);//找根 int v = get(e[i].v); if (u == v) continue;//continue 表示终止本次（本轮）循环，进入下一轮循环。判断它是否成环 fa[u] = v; cout << e[i].u << " " << e[i].v << endl; ans += e[i].w; } cout << ans << endl; return 0; }

拒绝采样

针对复杂问题的一种随机采样方法。 蒙特卡洛 蒙特·卡罗方法(Monte Carlo method)也称统计模拟方法，核心思想就是使用随机数（或更常见的伪随机数）来解决一些复杂的计算问题。 当所求解问题可以转化为某种随机分布的特征数（比如随机事件出现的概率，或者随机变量的期望值等）时，往往就可以考虑使用蒙特卡洛方法。通过随机抽样的方法，以随机事件出现的频率估计其概率，或者以抽样的数字特征估算随机变量的数字特征，并将其作为问题的解。这种方法多用于求解复杂的高维积分问题。 采样 在统计学中， 抽样（或称采样）是指从目标总体中抽取一部分个体作为样本的过程。 但是在计算机模拟时，我们所说的抽样，其实是指从一个概率分布中生成观察值（observations）的方法。而这个分布通常是由其概率密度函数（PDF）来表示的。而且，即使在已知PDF的情况下，让计算机自动生成观测值也不是一件容易的事情。从本质上来说，计算机只能实现对均匀分布（Uniform distribution）的采样。 PDF：概率密度函数（probability density function）, 在数学中，连续型随机变量的概率密度函数（在不至于混淆时可以简称为密度函数）是一个描述这个随机变量的输出值，在某个确定的取值点附近的可能性的函数。 PMF : 概率质量函数（probability mass function), 在概率论中，概率质量函数是离散随机变量在各特定取值上的概率。 CDF : 累积分布函数 (cumulative distribution function)，又叫分布函数，是概率密度函数的积分，能完整描述一个实随机变量X的概率分布。 一句话概括PDF就是f(x)，积分为1，与x轴面积=1，CDF是F(X),+∞时值为1 复杂问题用拒绝采样 找一个简单函数可以尽可能地罩住已知的。 原理 对所有x，有kq(x)≥f(x) 步骤 需要一个辅助的“建议分布”G（概率密度函数g(x)已知）来产生候选样本。可选均匀分布、正态分布等。 另一个辅助的均匀分布 u(0,1) 计算一个常数值 c,满足不等式cg(x)≥f(x)的最小值 开始生成样本 从G抽样得到Y 从U抽样得到V 若$V\le \frac{f(Y)}{C*g(Y)} $，则X=Y(即接受Y)，否则继续执行step1 重复直到样本总数够
