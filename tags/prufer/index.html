Prufer

树的计数相关问题

光看名字就比较高级了:D 学习这个的动力来自于一个问题：对于包含n个顶点的无向完全图最多包含$n^{n-2}$棵生成树。 Prüfer序列 Prüfer 序列可以将一个带标号 n 个结点的树用 [1,n] 中的 n-2 个整数表示。你也可以把它理解为完全图的生成树与数列之间的双射。常用于组合计数问题中。 对树建立序列 对树建立Prüfer序列的步骤：每次选择一个编号最小的叶结点并删掉它，然后在序列中记录下它连接到的那个结点。重复 n-2 次后就只剩下两个结点，算法结束。 先删除节点再记录标号，重复n-2之后只剩下根节点没删。 堆构造 听起来挺像建堆的过程，用堆的方法时间复杂度O(NlogN) 代码 // 结点是从 0 标号的 vector<vector<int>> adj;// adj的构造应该是：一行上每一个数表示和这行对应的点连接的节点序号 vector<int> pruefer_code() { int n = adj.size(); set<int> leafs; vector<int> degree(n); vector<bool> killed(n, false);//killed表示已经删除的 for (int i = 0; i < n; i++) { degree[i] = adj[i].size();//求每个点的度 if (degree[i] == 1) leafs.insert(i);//如果这行只有自己一个节点，就是度为0（degree=1）=叶子节点 } //求Prüfer code vector<int> code(n - 2); for (int i = 0; i < n - 2; i++) { int leaf = *leafs.begin();//叶子节点 leafs.erase(leafs.begin());//每次都会删掉一个节点 killed[leaf] = true; int v; for (int u : adj[leaf])//找leaf的邻接节点 if (!killed[u]) v = u; code[i] = v;//记录leaf的邻接节点 if (--degree[v] == 1) leafs.insert(v);//重新更新叶子节点 } return code; } 代码读懂了：一句话：按从小到大的顺序删除叶子节点，因为是树，code记录的就是被删除的叶子节点的parent 但是我不知道它有什么用.-. ...
