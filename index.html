自注意力机制

Self-Attention 是 Transformer最核心的思想 基础 公式 $$ Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}})V $$ 其本质是$softmax(XX^T)X$ XX^T 一个矩阵乘以它自己的转置，会得到什么结果，有什么意义？ 我们知道，矩阵可以看作由一些向量组成，一个矩阵乘以它自己转置的运算，其实可以看成这些向量分别与其他向量计算内积。向量的内积，其几何意义是什么？ 答：表征两个向量的夹角，表征一个向量在另一个向量上的投影 举例说明：$X = [x_1^T;x_2^T;x_3^T]$，模拟字向量的计算过程 行向量$x_i$分别与自己和其他两个行向量做内积（“早"分别与"上"“好"计算内积），得到了一个新的向量。 投影的值大，说明两个向量相关度高。 更进一步，这个向量是词向量，是词在高维空间的数值映射。词向量之间相关度高表示什么？是不是在一定程度上（不是完全）表示，在关注词A的时候，应当给予词B更多的关注？ 矩阵 $XX^T$是一个方阵，我们以行向量的角度理解，里面保存了每个向量与自己和其他向量进行内积运算的结果。 每一行可以看作是每个向量与所有向量的相关度 softmax $$ softmax(z_i)=\frac{e^{z_i}}{\sum_{c=1}^Ce^{z_c}} $$ 主要干的是归一化的工作，Softmax之后，这些数字的和为1了 为什么要softmax? 反方观点： 这篇用了线性attention，也就是去掉softmax，文章结论认为去掉softmax是可行的 这一篇也是线性attention 正方观点： attention的本质就是加权求和 softmax之后的数值就是权重 softmax(XX^T)X 从上文的$XX^T$继续看 一个行向量*X的一个列向量，得到： 在新的向量中，每一个维度的数值都是由三个词向量在这一维度的数值加权求和得来的，这个新的行向量就是"早"字词向量经过注意力机制加权求和之后的表示。 Q,K,T Q K V矩阵都是从X线性变化来的，为了提升模型的拟合能力 sqrt(d_k) 假设Q,K里的元素的均值为0，方差为1，那么$A^T=Q^TK$中元素的均值为0，方差为d. 当d变得很大时，A中的元素的方差也会变得很大，如果 A中的元素方差很大，那么softmax(A)的分布会趋于陡峭(分布的方差大，分布集中在绝对值大的区域)。 总结一下就是softmax(A)的分布会和d有关。因此A中每一个元素除以 $\sqrt{d_k}$后，方差又变为1。这使得softmax(A)的分布“陡峭”程度与d解耦，从而使得训练过程中梯度值保持稳定。 最后再补充一点，对self-attention来说，它跟每一个input vector都做attention，所以没有考虑到input sequence的顺序。更通俗来讲，大家可以发现我们前文的计算每一个词向量都与其他词向量计算内积，得到的结果丢失了我们原来文本的顺序信息。对比来说，LSTM是对于文本顺序信息的解释是输出词向量的先后顺序，而我们上文的计算对sequence的顺序这一部分则完全没有提及，你打乱词向量的顺序，得到的结果仍然是相同的。 这就牵扯到Transformer的位置编码了，具体在transformer里面看。 代码 不着急写，以后看

最小生成树

满足两点之间有且仅有一条简单路径的结构只有树 且这棵树包含 n 个节点。我们称这棵树为给定的图的生成树，其中总权值最小的生成树，我们称其为最小生成树。 最小生成树关键在避免环 生成树 定义 一个连通图的生成树是一个极小的连通子图，它包含图中全部的n个顶点，但只有构成一棵树的n-1条边。 性质 生成树的属性 一个连通图可以有多个生成树； 一个连通图的所有生成树都包含相同的顶点个数和边数； 生成树当中不存在环； 移除生成树中的任意一条边都会导致图的不连通， 生成树的边最少特性； 在生成树中添加一条边会构成环。 对于包含n个顶点的连通图，生成树包含n个顶点和n-1条边； 对于包含n个顶点的无向完全图最多包含$n^{n-2}$颗生成树。 证明$n^{n-2}$： 为什么是n^n-2），一篇里面塞不下:D 最小生成树 定义 所谓一个带权图的最小生成树，就是原图中边的权值最小的生成树 ，所谓最小是指边的权值之和小于或者等于其它生成树的边的权值之和。 求解 Kruskal 贪心找边，从小到大加入边 按边长度排序 从小到大，if加入e之后没有形成环，e加入G' 最后得到G‘ 问题是判断成环，见其它post Prim 加点，把点分成两坨A,B集合，A表示已经成了，B是待选的，要做的事从B里面挑一个点，满足它到A的点里面，有个边距离是最短的 （理解成长三角经济圈画好了，接下来先从周边里挑一个离长三角圈里面的城市最近的，这么想有点奇怪） 关键在怎么从待选里找距离最短的点 思路是用一个点个数大小的数组，没有直接连用∞，否则记录最小值 注意区分已访问过的点 这个为什么不会成环？把点分成两坨了，找的都是新点 示例与代码 测试数据 1 6 10 1 2 28 6 5 25 5 7 24 2 7 14 2 3 16 3 4 12 4 7 18 4 5 22 代码 使用Kruskal算法 #include<iostream> #include<algorithm> using namespace std; const int maxe = 100; int fa[maxe], n, m, ans = 0; struct edge { int u, v, w; }e[maxe]; bool cmp(edge a, edge b) { return a.w<b.w; } int get(int x)//用处是看它是否成环的，nb { if (fa[x] == x) return x;//x的根为它本身，x就是该集的根 return fa[x] = get(fa[x]);//路径压缩 } int main() { cin >> n >> m; for (int i = 1; i <= n; i++) fa[i] = i;//初始化 for (int i = 1; i <= m; i++) { cin >> e[i].u >> e[i].v >> e[i].w; } sort(e + 1, e + 1 + m, cmp);//升序排序 for (int i = 1; i <= m; i++) { int u = get(e[i].u);//找根 int v = get(e[i].v); if (u == v) continue;//continue 表示终止本次（本轮）循环，进入下一轮循环。判断它是否成环 fa[u] = v; cout << e[i].u << " " << e[i].v << endl; ans += e[i].w; } cout << ans << endl; return 0; }
