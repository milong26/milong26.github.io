Advances in Neural Information Processing Systems 36 (NeurIPS 2023)

https://proceedings.neurips.cc/paper_files/paper/2023/hash/fe692980c5d9732cf153ce27947653a7-Abstract-Conference.html

这篇论文探讨了模仿学习（Imitation Learning）中数据质量的重要性及其如何影响学习效果。模仿学习是让机器人通过观察专家演示来学习完成任务的一种方法，但一个常见问题是：学到的策略在真实测试时容易“跑偏”，因为小的错误会累积，导致机器人遇到从未见过的状态而失败。传统方法通常通过改进算法来解决这个问题，但本文提出，提升数据本身的质量可能同样甚至更有效。

论文的核心观点是，数据质量的好坏可以通过它是否能帮助学习策略在测试时“保持正轨”（即最小化分布偏移）来衡量。为此，作者提出了两个影响数据质量的关键属性：

  1. 动作分歧：指学习到的策略与专家策略在相同状态下做出的动作之间的差异。如果专家演示的动作不一致（比如专家操作时手抖或犹豫），或者学习算法难以准确模仿这些动作，就会增加动作分歧，导致测试时更容易出错。

  2. 转移多样性：指环境本身的不确定性或噪声。例如，同一个动作在执行时，由于物理世界的随机性（如摩擦力变化），可能导致不同的结果状态。

通过理论分析和在模拟环境（如点质量导航和机械臂操作任务）中的实验，论文得出几个重要结论：

  • 动作一致性至关重要：让专家的演示动作更一致、更确定（降低动作分歧），能显著提高学习策略的成功率。这意味着，一个动作干净利落的专家演示比一个犹豫或嘈杂的演示更有价值。

  • 状态多样性不是万能药：以往许多工作认为数据应尽可能覆盖更多不同的状态（即追求高状态多样性）。但本文发现，如果这种多样性是通过引入不稳定的专家动作（即增加动作分歧）换来的，反而会损害性能。有时，数据量少但动作一致的数据集，效果可能更好。

  • 系统噪声的双重作用：在环境中加入适量噪声（增加转移多样性）可以扩展数据的覆盖范围，让策略更鲁棒，甚至能部分抵消专家动作噪声带来的负面影响。但数据量如果太小，过多的噪声会导致数据过于稀疏，效果变差。

总的来说，这篇论文主张，在为模仿学习收集和整理数据时，不应只盲目追求“状态多样性”，而应更精细地平衡动作的一致性和环境的适度随机性，这样才能获得更高质量的数据集，从而训练出更可靠的机器人策略。

状态访问分布：智能体（不管是专家还是学到的策略）在执行完整任务过程中，所有可能访问到的状态构成的概率分布。这个分布是动态演变的。所以“匹配程度”比较的是两个策略（专家策略和学到的策略）各自引导出的整个状态轨迹的分布是否相似。从初始状态开始，在整个任务执行过程中，所有可能访问到的状态所构成的概率分布。它是一个动态的、由策略和环境共同决定的轨迹分布

动作分歧特指在同一个状态s下，学习策略的动作分布 π_A(a|s) 与专家策略的动作分布 π_E(a|s) 之间的差异（比如KL散度）。它不是直接比较两条不同的轨迹数据，而是比较在轨迹中可能出现的同一点上，两个策略会怎么做。这个差异是导致测试时状态分布逐渐偏离专家分布（即分布偏移）的直接原因之一。所以，动作分歧是发生在策略层面、导致最终问题的近因，而状态访问分布的匹配是我们要追求的最终结果。**“动作分歧”比较的就是：对于同一个状态 s，专家会怎么做，而学到的模型会怎么做。**​ 它衡量的是模型“模仿得像不像”的本质。

实验设计

1. 状态访问分布的匹配#

状态访问分布的匹配（matching of state visitation distribution）衡量的是学习到的策略在测试阶段访问到的状态分布是否和专家演示的状态分布相似。为了实现这一点，你需要定义“状态”以及如何度量状态的分布。

如何定义状态：原始图像 + 本体感觉 vs 冻结VLM输出#

在模仿学习中，状态通常由传感器数据（如图像、传感器读数等）组成。在你的实验中，你可以选择以下两种方式来定义“状态”：

  1. 原始图像 + 机器人本体感觉（raw image + robot proprioception）：这是最直接的状态表示方式，代表机器人在环境中的原始感知信息。这里，你需要保存机器人每一步的感知数据（如图像和本体感觉），然后通过这些信息来比较状态分布。

  2. 冻结VLM输出的内容（frozen VLM output）：如果你使用VLM（如GPT-4V）分析图像并从中提取特征，那么状态可以通过VLM输出的高层次表示来定义。冻结VLM的特征提取可以帮助简化原始图像的表示，并将其转换为更高层次、抽象的状态表示。

根据你的实验设置，如果VLM能够显著提取到图像中的重要特征，并且提高了机器人对环境的理解能力，冻结VLM输出的内容可能更合适，因为它将图像和本体感觉的高维信息压缩成更有用、更稳定的特征表示。

设计实验#

  • 对于专家示范数据和测试数据，分别计算它们的“状态访问分布”。你可以基于每一步的“状态”定义来构建状态分布。

  • 步骤：

    • 提取专家示范数据和测试数据中的状态（选择原始状态或VLM输出的状态表示）。

    • 计算状态的分布，常用的方法有统计频率分布、k-means聚类等。

    • 比较测试数据的状态分布与专家示范数据的状态分布的相似度，可以使用KL散度、JS散度或其他相似性度量方法。

  • 建议：如果你选择冻结VLM输出的内容作为状态，可以先用VLM对图像进行处理，得到每一步的抽象特征。然后基于这些高层次的表示来计算状态访问分布。这样，即使测试数据的图像和专家数据的图像在像素级上不同，你仍然可以从VLM的抽象表示中找到一致性。

2. 动作分歧#

动作分歧（action divergence）衡量的是在相同状态下，学习到的策略和专家策略之间的动作分布差异。你需要通过比较不同状态下的动作分布，来度量模型的模仿效果。

如何找到相同状态下的动作分布#

由于你提到每个“step”包含图像、本体感觉和动作，并且专家数据和测试数据的目标物体位置是随机的，如何定义相同的状态会是一个关键问题。因为像素级的图像不可能完全一致，所以我们需要通过特征层次上的“状态”来进行比较。

步骤：

  1. 定义状态：根据前面的分析，最好使用冻结VLM输出的高层次表示作为状态，而不是直接使用原始图像。这样可以避免因图像内容的随机差异而导致的匹配问题。

  2. 构建动作分布：

    • 每个状态（无论是原始状态还是VLM表示的状态）对应一个动作分布。在专家数据和测试数据中，记录每一步的动作。

    • 对于专家数据，你有288集数据，每集包含若干step。在每个step中，你可以得到一个动作（假设动作是离散的或通过某种方式量化的）。

    • 对于测试数据，同样记录每一步的动作。

    • 在同一个状态下，专家策略的动作分布和测试策略的动作分布可以通过统计方法（如直方图、KL散度等）进行比较。

  3. 比较动作分布：

    • 比较每个状态对应的动作分布（专家和测试数据），并计算它们之间的差异。常用的度量方法有KL散度或JS散度，这两种方法都可以衡量两个分布的差异。

    • 需要注意的是，比较动作分布时，你可能无法做到完全一对一地对齐每个状态和动作，因此可以采用聚合的方式来对比整体分布差异，而不是单一的step。

实验设计：#

  • 通过VLM的输出对每个状态进行特征提取。

  • 对每一集（或每一步）记录对应的动作，并计算每个状态下的动作分布。

  • 比较专家数据和测试数据在相同状态下的动作分布，使用KL散度等方法量化差异。

分析粒度：你可以选择按“集”为单位来计算动作分布的差异。也就是说，比较一集数据中的状态分布和动作分布，而不是逐步对齐每个step的动作分布。这样可以避免由于小的时序差异导致的干扰，同时保持数据分析的合理性。

3. 实验步骤总结#

  • 状态访问分布匹配：

    • 提取专家示范数据和测试数据的状态表示（使用原始状态或冻结VLM输出的状态）。

1. 状态相似性的计算方法与伪代码#

论文中“状态相似性”（State Similarity）是衡量状态一致性的指标，其值越低，表示状态多样性越高。它被定义为数据集中所有状态的“邻居”状态数量的平均值。

该指标的计算方法在**附录B（第18页）**​ 有明确定义。其核心思想是：对于数据集中的每一个状态 s，定义一个距离阈值 ε，找出数据集中所有与 s的距离小于等于 ε的其他状态，形成一个“聚类”（或邻居集合）。状态相似性就是这个聚类大小的期望值。

公式定义 (Equation 10):

StateSimilarity(D) = (1/|D|) * Σ_{s,a ∈ D} |C(s, D)|

其中：

  • D是整个数据集。

  • |D|是数据集的大小（状态-动作对的数量）。

  • C(s, D) = { (s̃, ã, s̃’) ∈ D : ||s - s̃|| ≤ ε }，即数据集中所有与状态 s的距离在 ε范围内的数据点集合。

  • |C(s, D)|是该集合的大小（即 s的“邻居”数量）。

伪代码表示:

def calculate_state_similarity(dataset D, distance_threshold epsilon):
    """
    计算数据集D的状态相似性。
    参数:
        D: 数据集，每个元素为 (state, action, next_state) 元组。
        epsilon: 距离阈值，用于判断两个状态是否“相似”。
    返回:
        state_similarity: 浮点数，状态相似性值。
    """
    total_similarity = 0
    N = len(D)  # 数据集大小

    for i in range(N):
        s_i, _, _ = D[i]  # 当前查询状态
        neighbor_count = 0

        # 遍历整个数据集，计算邻居数量
        for j in range(N):
            s_j, _, _ = D[j]  # 候选状态
            if distance(s_i, s_j) <= epsilon:  # 使用合适的距离度量（如L2范数）
                neighbor_count += 1

        total_similarity += neighbor_count

    state_similarity = total_similarity / N
    return state_similarity

2. 核心评估指标#

结合论文中的思路，以下是适应VLM的指标和评估方法：

a. 任务特征空间的覆盖（Semantic Coverage）#

这与原论文中的“状态覆盖”类似，但要考虑的是VLM输出的语义特征空间，而不是物理状态空间。

  • 方法：

    1. VLM聚类分析：使用VLM（如GPT-4V）对专家示范进行处理，将每个状态映射到VLM特征空间中。然后，进行聚类分析，看看任务的关键语义特征是否得到充分覆盖。

    2. 分析聚类中心：检查VLM特征空间中的聚类中心的数量和分布，确保数据集覆盖了任务所需的所有关键动作模式（例如：抓取、移动等）。

    3. 覆盖度指标：通过聚类数量、覆盖率等指标，定量化任务关键特征的覆盖情况。

  • 示例评估：假设任务是“抓取物体”，你可以计算VLM特征空间中聚类的数量和密度，确保任务所需的动作（如抓取、稳定物体）得到了足够的表示。

b. 动作一致性与质量（Action Consistency and Quality）#

这里不再直接计算“动作分歧”在物理状态空间中的差异，而是评估专家示范在语义空间中的一致性和质量。

  • 方法：

    1. 任务阶段划分：使用VLM或其它方法对专家演示进行任务阶段分割（例如：“接近物体”，“抓取物体”）。然后，评估每个阶段内的动作一致性。

    2. 动作方差：在每个任务阶段内，计算专家动作的方差。如果同一任务阶段内的动作分布非常分散，可能说明该阶段的专家示范不够一致，导致训练模型可能难以模仿。

    3. 专家动作质量评估：可以通过VLM的推理能力直接评估动作的流畅性、精确性和一致性。例如，使用VLM（如GPT-4V）对专家示范的视频片段进行分析，评估动作的自然性与准确性。

  • 示例评估：假设在抓取任务中，专家示范在“接近物体”阶段的动作一致性较差，VLM可以通过分析视频片段，找出是否存在不一致的动作（例如，专家的手抖动、迟疑）。你可以计算该阶段的动作方差，并对高方差阶段的数据进行筛选或改进。 搜索有没有 评估 vla数据
