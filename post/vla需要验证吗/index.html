看这篇文章的目的：为什么不用验证？

数据之间的分布

验证的目的：在经典的监督学习中，在训练过程中通常会计算分类精度或在保留的验证集上回归的均方误差等指标。这些指标是模型能力的强指标，可以概括为在与训练集相同的上下文中收集的不可见输入。因此，它们用于选择最佳模型检查点或“提前停止”训练。

然而，在现实世界机器人的端到端模仿学习的背景下，对于使用验证集来选择最佳检查点的最佳度量和实践，专家们并没有明确的共识。这是因为机器人专家旨在优化的指标是成功率。换句话说，机器人完成任务的成功尝试百分比。它需要在测试环境中的机器人上长时间运行策略，以确保外部因素引起的低方差。例如，光线条件可能会发生变化，房间布局可能会每天发生变化，机器人电机的动态可能会因使用而发生变化等。只能计算验证损失或其他代理度量，例如动作空间中的均方误差。

由于计算每个模型检查点的成功率成本太高，一些专家建议使用验证损失来选择最佳检查点。例如，ACT和Aloha作者Zhao et. al.指出“在测试时，我们加载实现最低验证损失的策略，并将其在环境中推出”。另一方面，斯坦福大学Robomimic的作者注意到“当我们评估所有检查点时，最好的验证丢失时提前停止的策略比性能最好的策略差50%到100%"，这表明选择验证丢失最低的检查点并不能确保最高的成功率。

一些假设可以解释为什么低验证损失不能预测高成功率。首先，在通过人类遥操作训练期间收集的数据与通过控制机器人的策略在评估期间获得的数据之间的分布可能会发生变化。这种转变可能是由于我们之前列出的环境或机器人的所有可能变化，也可能是由于策略的轻微预测误差随着时间的推移而积累并将机器人移动到常见轨迹之外。因此，policy在评价期间看到的input可能与训练期间看到的input大不相同。在这种情况下，计算验证损失可能没有帮助，因为损失函数用于优化人类演示轨迹的复制，但它没有考虑到在训练分布之外泛化的能力。它不会直接优化在可能变化的环境中完成任务的成功率。

在这项研究中，我们将探讨验证损失是否可以用来选择与最高成功率相关的最佳检查点。如果事实证明并非如此，那么在训练集的保留子集上计算验证损失可能是无用的，甚至可能损害性能，因为训练是在训练集的较小部分上完成的。我们还将讨论使用验证损失的替代方案。我们的实验是在两个常用的模拟环境中进行的，PushT和Aloha Transfer Cube，有两种不同的策略，分别是扩散和ACT（Action chunking with transformers）。模拟使我们能够准确地计算每10K个检查点的成功率，这在前面解释的真实的环境中是具有挑战性的。

  • 传统的监督学习可以用验证集来证明模型对未见过的输入的泛化性，但是机器人为什么不可以？哪怕是相同的物体不同的位置
  • 机器人action通过真实移动感知世界，这一个怎么建模？

在经典监督学习里，验证集的核心作用有三个： （1）检测过拟合（overfitting） 在理想情况下： 训练 loss：随着训练步数增加，单调下降或趋于平缓 验证 loss：先下降（学到有用规律），后上升（开始记忆训练集噪声） 于是就出现了经典图像： “训练 loss 继续下降，但验证 loss 反弹” 这意味着模型开始记忆训练数据的细节，而不是学习可泛化的规律。

（2）验证集 ≈ “真实测试分布的抽样”#

传统监督学习隐含了一个极其重要、但常被忽略的前提：

训练集、验证集、测试集是 i.i.d.（独立同分布）采样的 也就是说：

  • 数据来自同一个生成机制
  • 输入分布 p(x) 没变
  • 标签函数 p(y∣x)没变 在这个前提下：

在验证集上 loss 低
⟹ 在未来未见数据上 loss 也低（高概率） 这是验证集“能代表泛化性”的理论根基。

（3）监督学习的目标函数 = 评估指标（或高度相关）#

再强调一个关键点：

  • 训练目标：最小化 CE / MSE
  • 验证指标：CE / MSE / Accuracy
  • 测试指标：Accuracy / RMSE 它们是同一个问题的不同表达。 所以验证 loss 不只是“相关”，而是直接优化目标的代理。 机器人模仿学习中，验证集并不能代表模型在执行时看到的数据分布 就是说，机器人模仿学习中，最开始的输入还可能代表模型在执行时看到的数据分布，但随着机器人执行动作，改变了 这是一个策略会影响输入的时序变化的过程。 而且最终的成功率也是非连续指标，只有成功和失败 pusht实验里面的图里的 standard deviation 是什么？ 图里面的每个点（validation loss / MSE / success rate）不是单次训练的结果，而是：

从刚才为什么vla模仿学习不能验证延伸出两个问题：

  1. 第一个输入都是iid的，这个时候应该可以验证。但是第一个动作的误差，和最终失败之间是非线性的
  2. 如果把指标改成连续的，从成功/失败改成相差的欧式距离？ 但就算这样，你在验证集上测到的，是“在 expert 分布下的末端误差”，不是“在 policy-induced 分布下的末端误差”。因为输入数据是人类采集出来的。 如果我想验证，需要的是输入图片-输出距离的pair，而不是动作的。动作本质上是需要和环境交互。

在端到端模仿学习（尤其是 VLA）中，由于策略在执行时会诱导新的状态分布，验证集上的 imitation loss 或其连续代理指标通常无法可靠地预测真实任务成功率，因此不适合作为 early stopping 的主要依据。

在真实的世界中，评估具有低方差的给定检查点的成功率是极其昂贵的。当然，在训练时不可能在每个检查站都这样做。相反，我们建议进行一些评估，主要集中在定性评估上，例如学习新功能和机器人运动的流动性。当没有更多的进展时，可以停止训练。

例如，当训练PollenRobotics的Reachy 2（参见演示）抓住一个杯子并将其放在架子上，然后抓住一个苹果并将其交给坐在对面的人的手，并旋转回初始位置时;我们注意到该策略逐渐学习了更先进的概念和轨迹：

  • 在检查点20k，机器人只能抓住杯子，但无法将其放在架子上。
  • 在检查点30k，它学会了平稳地把苹果放在架子上，但没有抓住苹果。
  • 在检查点40 k，它学会了抓住苹果，但没有旋转。
  • 在第50 K点，它学会了旋转并把苹果给你，但它没有旋转回来。
  • 最后，它学会了旋转回到所需的最终位置，并完成完整的轨迹。

频繁进行小型定性评估是一种有效的方法，可以发现错误，感受从一个检查点到其他检查点的策略功能和稳定性，并获得改进方法的灵感。

总结这篇：

  • 验证集指标在端到端机器人模仿学习中不可靠
    • 传统监督学习中，验证集用于 early stopping 依赖两个假设：
      1. 验证集与执行时的输入分布相同
      2. 训练目标与最终评估指标一致
    • 在机器人模仿学习中：
      • 策略执行会改变状态分布（policy-induced distribution shift）
      • 训练损失（imitation loss / MSE）与任务成功率（success rate）不对齐
        → 因此，验证集无法用来预测最终任务成功率，也不适合做 early stopping。
  • 直接找到最优 checkpoint 在真实机器人上非常昂贵
    • 需要大量 rollout 才能获得低方差的 success rate
    • 中间频繁评估成本高，且耗时长
      → 单纯依赖“测所有 checkpoint”在真实环境中不可行。
  • 可行的训练与停止策略
    • 仿真先行 + sim-to-real
      • 在高保真仿真中评估策略的 success rate
      • 选取表现最好的 checkpoint，再迁移到真实机器人
    • Capability Milestone Stopping（能力里程碑停止）
      • 将任务拆解为关键子能力（milestone）
      • 定期在训练中观察模型是否学会这些子能力
      • 当所有里程碑能力学会且稳定时，训练可停止
      • 结合 qualitative 观察，判断动作平滑性和鲁棒性 模糊的概念
  1. validation适用于多任务吗？ 不一定，我感觉悬
  2. vla的重点在输入还是输出后的交互？有没有办法能把交互建模到一个比较好解决的问题
  3. 我之前设想过一个方案，用于实现真正的泛化的，机器人先自行移动以感知空间，这样就不需要进行单任务收集数据微调这样反复操作。而是把vla转换成空间问题，但只是模糊的概念 前两个不太有用，第三个 自主探索 + 泛化训练：
  • 机器人在真实环境中先随机或策略探索动作
  • 收集状态-动作对，形成丰富空间数据
  • 训练 VLA / policy，使其能够在任意空间位置完成任务，而不是针对单任务微调

优势#

  1. 减少单任务微调的重复成本
  2. 增加状态覆盖率 → 提升泛化能力
  3. 可以把任务目标转化为“空间目标误差”，变成连续优化问题
    • 比 success/failure 更连续
    • 对 early stopping /训练监控更友好

挑战#

  • 如何保证探索安全、有效（碰撞 / 不破坏环境）
  • 需要策略能在高维感知空间中快速学习（sample efficiency）
  • 还要考虑long-horizon compounding error

实际上，这是一条接近 自主探索 + model-based / goal-conditioned policy 的路线。
论文中叫做 self-supervised or exploration-driven VLA。 让机器人通过自主探索 + 空间建模 + 连续目标学习获得泛化能力，把 VLA 问题从单任务离散成功率问题转化为连续空间优化问题 一个不知道能不能做的切入点： 在小范围内（仿真环境），Loss：最终物体位置 vs 目标位置的相机坐标误差（连续指标） 强化学习应该做过类似的东西吧？？ 先留着，暂时不再深入
