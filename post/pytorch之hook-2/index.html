pytorch之hook

December 11, 2023 · map[avatar:images/avatar.jpg bio:Make milong famous again links:[map[name:GitHub url:https://github.com/milong26]] name:Milong]

hook，钩子。一般来说，“hook”是在特定事件之后自动执行的函数。

参考pytorch-hook文档 前情提要

x = torch.tensor([2.], requires_grad=True)
a=x**2
b=a**3
b.backward()
# 求grad和back是在一起的，算出来就是db/dx
print(x.grad)

定义#

PyTorch提供四种Hook函数： 1、torch.Tensor.register_hook(hook) 2、torch.nn.Module.register_forward_hook 3、torch.nn.Module.register_forward_pre_hook 4、torch.nn.Module.register_backward_hook

torch.Tensor.register_hook#

功能：注册一个反向传播hook函数，Hook函数仅一个输入参数，为张量的梯度。Hook不应修改其参数梯度值，但可以选择返回一个新的梯度，该梯度将代替grad使用。

hook(grad) -> Tensor or None

def hook_grad(grad):
    print(grad)

x = torch.tensor([1.], requires_grad=True)
a=x*2
b=a**2
c=b**3
handle = b.register_hook(hook_grad)
c.backward()
print("x.grad:",x.grad)

在hook中输出的grad为dc/dc(backward/register) 最后输出的x.grad为dc/dx(backward/requires_grad=True) 更多内容看autograd

torch.nn.Module.register_forward_hook#

功能：注册module的前向传播Hook函数 参数：

  • module：当前网络层
  • input：当前网络层输入数据
  • output：当前网络层输出数据
import torch
import torch.nn as nn

# 构建网络，一个卷积层一个池化层
class Net(nn.Module):
    def __init__(self):
        super(Net,self).__init__()
        self.conv1 = nn.Conv2d(1, 2, 3)
        self.pool1 = nn.MaxPool2d(2)

    def forward(self, x):
        # 卷积
        x = self.conv1(x)
        # 池化
        x = self.pool1(x)
        return x
# 初始化网络
net = Net()
# detach将张量分离
# fill_将tensor中的所有值都填充为指定的value
# conv1的weight有2张，都是3x3，第一张全填充1，第二张全填充2
net.conv1.weight[0].detach().fill_(1)
net.conv1.weight[1].detach().fill_(2)
net.conv1.bias.detach().zero_()
print(net.conv1)
print(net.conv1.weight)
print(net.conv1.bias)
# fake_img bacth_size=1,input_channel=1,为4x4大小，每个格子都是1的二维
fake_img = torch.ones(1,1,4,4)

# 定义forward_hook
# input：当前网络层输入数据
# output：当前网络层输出数据
def forward_hook(module, data_input, data_output):
    print("output",data_output)
    print("input",data_input)

# 注册Hook
net.conv1.register_forward_hook(forward_hook)
output = net(fake_img)

最后显示的是卷积层进行了一次卷积的输入和输出。

torch.nn.Module.register_forward_pre_hook#

功能：注册module前向传播前的hook函数。 参数：

  • module：当前网络层
  • input：当前网络层输入数据

torch.nn.Module.register_backward_hook#

功能：注册module反向传播的hook函数。 参数： module：当前网络层

  • grad_input：当前网络层输入梯度数据
  • grad_output：当前网络层输出梯度数据

网络#

PyTorch 为每个张量或 nn.Module 对象注册 hook。hook 由对象的向前或向后传播触发。它们具有以下函数签名:

from torch import nn, Tensor

def module_hook(module: nn.Module, input: Tensor, output: Tensor):
    # For nn.Module objects only.
    
def tensor_hook(grad: Tensor):
    # For Tensor objects only.
    # Only executed during the *backward* pass!

每个 hook 都可以修改输入、输出或内部模块参数。最常见的是用于调试目的。但我们将看到它们还有很多其他用途。

应用#

Hooks for saved tensors#

定义一组pack_hook / unpack_hook来控制已经保存的张量要不要被打包

这里打包指的是：为了防止引用循环，PyTorch在保存时将张量打包并将其解包到不同的张量中以供read 张量是否会被打包到不同的张量对象中取决于它是否是自己的grad_fn的输出 grad_fn需要参考autograd

  • pack_hook函数 接受一个张量作为其单个参数 可以返回任何python对象（例如另一个张量，元组，甚至包含文件名的字符串）
  • unpack_hook函数 将pack_hook的输出作为其单个参数 返回一个用于向后传递的张量 unpack_hook返回的张量只需要与作为输入传递给pack_hook的张量具有相同的内容。 特别是，任何与autograd相关的元数据都可以忽略，因为它们将在解包期间被覆盖。

举例：

class SelfDeletingTempFile():
    def __init__(self):
        self.name = os.path.join(tmp_dir, str(uuid.uuid4()))

    def __del__(self):
        os.remove(self.name)

def pack_hook(tensor):
    temp_file = SelfDeletingTempFile()
    torch.save(tensor, temp_file.name)
    return temp_file

def unpack_hook(temp_file):
    return torch.load(temp_file.name)

注意： pack_hook只会被调用一次，但是unpack_hook会调用很多次，所以这个函数不能删除临时文件
