最小生成树

September 15, 2023

满足两点之间有且仅有一条简单路径的结构只有树 且这棵树包含 n 个节点。我们称这棵树为给定的图的生成树，其中总权值最小的生成树，我们称其为最小生成树。 最小生成树关键在避免环

生成树#

定义#

一个连通图的生成树是一个极小的连通子图，它包含图中全部的n个顶点，但只有构成一棵树的n-1条边。

性质#

生成树的属性 一个连通图可以有多个生成树； 一个连通图的所有生成树都包含相同的顶点个数和边数； 生成树当中不存在环； 移除生成树中的任意一条边都会导致图的不连通， 生成树的边最少特性； 在生成树中添加一条边会构成环。 对于包含n个顶点的连通图，生成树包含n个顶点和n-1条边； 对于包含n个顶点的无向完全图最多包含$n^{n-2}$颗生成树。 证明$n^{n-2}$： 为什么是n^n-2），一篇里面塞不下:D

最小生成树#

定义#

所谓一个带权图的最小生成树，就是原图中边的权值最小的生成树 ，所谓最小是指边的权值之和小于或者等于其它生成树的边的权值之和。

求解#

Kruskal#

贪心找边，从小到大加入边

  1. 按边长度排序
  2. 从小到大，if加入e之后没有形成环，e加入G'
  3. 最后得到G‘ 问题是判断成环，见其它post

Prim#

加点，把点分成两坨A,B集合，A表示已经成了，B是待选的，要做的事从B里面挑一个点，满足它到A的点里面，有个边距离是最短的 （理解成长三角经济圈画好了，接下来先从周边里挑一个离长三角圈里面的城市最近的，这么想有点奇怪） 关键在怎么从待选里找距离最短的点 思路是用一个点个数大小的数组，没有直接连用∞，否则记录最小值 注意区分已访问过的点 这个为什么不会成环？把点分成两坨了，找的都是新点

使用Kruskal算法

#include<iostream>
#include<algorithm>
using namespace std;
const int maxe = 100;
int fa[maxe], n, m, ans = 0;
struct edge
{
	int u, v, w;
}e[maxe];

bool cmp(edge a, edge b)
{
	return a.w<b.w;
}
int get(int x)//用处是看它是否成环的，nb
{
	if (fa[x] == x) return x;//x的根为它本身，x就是该集的根
	return fa[x] = get(fa[x]);//路径压缩
}
int main()
{
	cin >> n >> m;
	for (int i = 1; i <= n; i++)
		fa[i] = i;//初始化
	for (int i = 1; i <= m; i++)
	{
		cin >> e[i].u >> e[i].v >> e[i].w;
	}
	sort(e + 1, e + 1 + m, cmp);//升序排序
	for (int i = 1; i <= m; i++)
	{
		int u = get(e[i].u);//找根
		int v = get(e[i].v);
		if (u == v) continue;//continue 表示终止本次（本轮）循环，进入下一轮循环。判断它是否成环
		fa[u] = v;
		cout << e[i].u << " " << e[i].v << endl;
		ans += e[i].w;
	}
	cout << ans << endl;
	return 0;
}
