placo文档 mplib文档

特性 Placo MPlib
主要功能 机器人任务空间逆运动学 (IK) / 逆动力学 (ID) 及运动规划接口 轻量级运动规划库，包含 IK 支持，但更偏向路径规划和碰撞避让
依赖库 Pinocchio（高性能动力学）、eiquadprog（QP 求解器） Pinocchio 或 KDL（动力学）、OMPL（运动规划）、FCL/CCD（碰撞检测）
层级关系 高级 IK/ID API（封装了 Pinocchio + QP 优化） 高级运动规划 API，内部可调用 Pinocchio/KDL 做 IK，主要针对路径规划
系统依赖 独立 C++ 库 + Python 绑定，可嵌入控制系统 C++ 核心库 (libmp.a) + Python 绑定 (pymp.so)，可在模拟器（Sapien/MuJoCo）中直接使用
典型用途 需要任务空间约束 IK/ID、整机控制、MPC、强化学习 快速实现机器人路径规划、抓取、避障、可在 Python 中实验
  • Placo 更偏向 控制层/IK求解器层，MPlib 更偏向 运动规划/路径规划层。
  • 在 IK 方面，MPlib 会调用 Pinocchio/KDL，但没有像 Placo 那样提供丰富的任务空间约束和正则化选项。

在ik方面：

维度 Placo MPlib
IK 精度与约束 高，可加权约束和正则化 中，主要依赖底层 Pinocchio/KDL
奇异点处理 有办法通过正则化/优化缓解 无内置处理，需要自己实现
关节翻转 / 临界值 可以通过任务空间约束和正则化缓解 可能出现翻转，需要路径规划或后处理
性能 高性能，专为 IK/ID 设计 高性能，但 IK 是规划的一部分，功能不如 Placo 完整

  1. Regularization：缺少正则化导致 IK 解跳到另一个等效姿态分支。
    1. 怎么加？solver
    2. 有哪些正则化？优点缺点？默认的是哪一个？
    3. 正则化在sovler计算ik的时候起到一个什么作用？本来ik是一个欠约束问题，Regularization 的数学作用是向求解器加了一个轻微的惩罚项，对关节变化量 $\Delta q$ 做轻微惩罚。默认使用的是L2正则化，惩罚系数非常小，求解器的主要目标仍然是 末端任务精度
名称 效果（作用） 优点 缺点 典型使用场景
默认 L2 正则化（Always on） 最小化 ‖Δq‖²，使解尽量接近当前关节配置 免费、自动、有助避免跳变 默认权重很小，无法避免严重翻转 所有 IK 求解中自动存在，但不能单独依赖
Custom L2 RegularizationTask 强化“不要远离上一姿态”，对所有关节均匀抑制大变化 最有效的防跳变方式；越大越稳定 太大会导致目标追踪滞后 防止 IK 解翻到另外一个分支）
Posture Regularization (JointsTask) 让关节更靠近“某个姿态”（通常是零位或中性位） 防止解漂到奇怪的姿态区域 如果参考姿态不合理，可能引导到错误区域 解决冗余手臂漂移、靠近某个全局参考姿态
Kinetic Energy Regularization 最小化(1/2) q̇ᵀ M q̇，抑制高加速度 控制动态、抑制抖动、避免瞬间翻转 依赖 dt，对静态 IK 效果小 时间连续轨迹中保持平滑、避免短时间抖动
Manipulability Regularization 增大可操作度，避免进入奇异点 优化姿态质量、降低奇异点不稳定性 不能单独解决翻转，只改善奇异点附近行为 接近奇异点时增强稳定性（但不会防跳变）
