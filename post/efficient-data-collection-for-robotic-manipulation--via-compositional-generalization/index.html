  15. 为简化问题，我们假设该元组的每个元素都有若干个离散值可选。比如，机器人可能遇到的物体类型有若干种。我们将这些离散值称为“因素值”。因此，该元组的每个元素都代表了环境中各种因素值的组合，比如机器人可能遇到的特定物体类型和桌子高度。 p⁢(f) 表示各种因素值组合的分布情况，而 ρ⁢(s0|f) 则是在给定某些因素值组合的条件下，系统的初始状态分布。我们采用这种表述方式来说明：环境中的各种因素值可以通过影响系统的初始状态，从而影响任务的执行结果。同样地， μ⁢(f):ℱN→S 是一个确定性函数，它将各种因素值组合映射到期望的目标状态 g∈S 上。因为因素值决定了任务的目标状态，所以 μ⁢(f):ℱN→S 的作用就是确定目标状态。我们的目标是学习一种以目标为条件的策略 π(⋅|s,g):S×S→Δ(𝒜) ，其中 Δ⁢(𝒜) 表示动作空间上的概率分布。我们的目标是在各种可能的因素值分布下，最大化达到期望目标的概率。差不多，因素设计就是给环境进行人为的物理分类

总结：这篇论文的核心问题是：机器人模仿学习策略能不能像“拼积木”一样，把训练数据里学到的不同环境因素组合起来，去处理没见过的组合场景，以及如果能做到，怎样的数据收集方式更省力。论文明确说他们做了模拟环境和真实机器人的系统实证研究，来检验 visual imitation learning policies 是否能对环境因素做组合泛化

  • to： 研究视觉模仿学习策略能否对环境因素做组合泛化，并据此设计更高效的机器人 in-domain 数据收集方式，以减少收数成本。
  • this work：创新点而是系统分析环境因素的组合效应，提出利用组合泛化来指导 in-domain data collection 的策略（如 Stair、L 等）。
  • get： 在真实机器人上，基于这种策略训练的最好模型在转移到全新环境时达到了 77.5% 的成功率，而没有 prior data 时只有 27.5%，没有环境变化的 in-domain 数据时只有 2.5%。
  • but： 局限性是，论文主要考虑的是少量任务和一个 prior dataset，且对物理因素之间的组合仍然较难，作者也承认未来需要扩展到更多机器人平台、更多任务以及更大规模的数据收集。

真实机器人实验 在真实厨房场景验证组合泛化是否成立，并测试 prior data 的作用 89 使用 WidowX 250 6DOF robot arm；任务是 putting a fork inside a container；主要因素是 object type、object position、container type、table height、table texture，每个因素有 k = 4 个值；只做 N = 5 的设置；主要比较 Stair 和 L，再加 No Variation；每个策略收集 16 个组合、每个组合 10 条演示，总共 160 条演示；数据都在 BaseKitch 这个厨房里收集；同时比较 from scratch 和 BridgeData V2 作为 prior data 的效果 101112 验证组合泛化能否迁移到真实机器人，尤其看 prior robot data 是否能增强组合能力，以及哪些因素更容易组合、哪些更难 13
额外真实实验 测试 frozen visual representations 对组合泛化是否有效 14 额外评估 R3M 和 VC-1 作为 frozen visual encoders，在部分 factor pairs 上测试；关注 object type + table texture、object position + table texture、object position + table height 这些组合 151617 验证现代 frozen visual representations 是否也能支持真实机器人上的组合泛化；作者的结果是它们表现不好 18
