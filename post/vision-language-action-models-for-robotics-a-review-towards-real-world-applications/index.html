  • 本综述与以往综述的区别（本综述的贡献点）

  1. vla和别的？？？的区分有助于理解定义
  2. 来源：LLM 和 VLM 用于解释多模态输入、推理任务并执行上下文感知action
  3. 发展轨迹
    1. 早期将 LLM 和 VLM 与负责作生成的底层机器人策略解耦：机器人策略（Policy Module）：根据前两步的输出（文本描述 + 感知信息），调用预定义的动作集合或者模仿学习，所以没法推广到更加广泛的任务。它的难点是没法拓展到unseen task
    2. 后来的研究（VLA模型）把视觉、语言和动作放在一个端到端（end-to-end）的模型里联合学习（joint learning），关键词变成了
  4. 本文内容和其他总数的区别
  5. 重新定义了VLA：A Vision-Language-Action (VLA) model is a system that takes visual observations and natural language instructions as required inputs and may incorporate additional sensory modalities. It produces robot actions by directly generating control commands. Thus, models in which a high-level policy (e.g., a vision-language model backbone) merely selects an index from a set of pre-trained skills or control primitives are excluded from this definition.一个视觉+自然语言指令（和其它模态）作为输入，直接生成控制命令。不能是那种选择动作原语的模型，那位姿也可以啊
  6. 介绍结构，挑战一般就是问题了，策略、架构、训练分别对应：宏观策略 → 中观架构 → 微观训练。

    1. VLA 要解决什么问题
      1. 我希望机器人能做什么？（任务目标）
      2. 当前系统为什么做不到？（痛点）
      3. 我的VLA系统能在哪些方面改进现有方案？（贡献点）
    2. 怎么设计思路去解决，也就是一个总体策略
      1. 模块化设计还是端到端设计？
      2. 强调通用性（generalist）还是任务特定优化？
      3. 强调大规模预训练还是数据高效学习？
      4. 用哪种范式（transformer-based? diffusion-based? RL-based?）
    3. 系统内部怎么实现这种思路，用什么模块
      1. 视觉编码器（Vision Encoder）：提取环境视觉特征。
      2. 语言编码器（Language Encoder）：解析任务命令。
      3. 多模态融合模块（Fusion Module）：让视觉、语言信息对齐。
      4. 动作生成模块（Action Decoder / Policy Head）：输出动作指令或控制信号。
      5. 记忆与状态模块（Memory / Context）：处理时序与上下文信息。
    4. 模型具体怎么训练和运行：我打算用什么学习机制，让模型学会执行任务？
        • 监督学习（supervised learning）：从配对数据中学习。
      2. 模仿学习（imitation learning）：学习人类演示的动作。
      3. 强化学习（reinforcement learning）：在交互中学习策略。
      4. 自监督预训练（self-supervised pretraining）：用未标注数据学习模态间对齐。
      5. 多阶段训练：预训练 + 微调（pretrain + fine-tune）。
    5. 评估：“我的系统在真实世界中表现如何？”
      1. 定义指标
      2. 选择平台

挑战#

VLA 要解决什么问题

数据#

训练 VLA 模型需要“大规模、多样化、三模态（视觉、语言、动作）对齐”的数据集。

跨机器人迁移#

  1. 形态差异大，（action space）和感知空间（observation space）不同
  2. 策略（policy）也难直接迁移到其他机器人
  3. 现实数据和机器人行动有形式（form）和语义（semantics）上的差别，所以不能直接迁移

计算成本#

  1. Transformer计算复杂->预训练VLM
  2. 仍极其耗费算力，尤其在处理长时序、高分辨率或包含3D、身体感知等额外模态时更为严重
  3. 部署有问题 因此需要研究更高效的模型结构与蒸馏方法来降低资源需求
