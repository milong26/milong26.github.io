  1. MinGW

    链接

    往下滑，找到最新版的 “x86_64-posix-seh”。

    下载后得到一个7z压缩包，解压到指定conf位置D:\conf\MinGW

    编辑用户变量Path，加入刚才的D:\conf\MinGW\mingw64\bin

    验证：g++，提示：g++: fatal error: no input files，证明🆗

  2. 配置vs的c++环境（和以前在ubuntu上面有点像，所以很痛苦的，是一个文件夹里配置）

    打开一个文件夹，新建一个cpp文件，写点代码上去。

    侧边工作栏的bug按钮，（以下最好连续）**选中Run and Debug，选C++(GDB/LLDB)，g++.exe，**会自动生成一个launch.json.

    然后咱们来看看launch.json啊

    {
    "version": "0.2.0",
    "configurations": [
        {
            "name": "g++.exe build and debug active file",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}\\${fileBasenameNoExtension}.exe",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": true,      //修改此项，让其弹出终端
            "MIMode": "gdb",
            "miDebuggerPath": "D:\\conf\\MinGW\\mingw64\\bin\\gdb.exe",
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "task g++" //修改此项
        }
    ]
}

    保存，返回c++，停止运行，按F5调试，提示找不到文件。选择Configure Task，编辑tasks.json，把其中label行修改成 "label": "task g++",

    （ps它有好几个选项，不知道选哪个，姑且改了第一个，有问题以后再改。）

    再F5调试，ok
