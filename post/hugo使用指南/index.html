安装#

下载v0.156.0release搜索到有windows版本的，最好直接安装extended版本：hugo_extended_withdeploy_windows命名的文件下载 设置环境变量，Path中添加hugo解压后的文件夹（可以访问到hugo.exe的那个路径） hugo version检查有结果了就行

使用前的配置#

在期望存放的位置

hugo new site guance

出现Congratulations! Your new Hugo project was created in表示成功 进入文件夹，必须先找到一个主题，用PaperMod吧

git init  
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

然后修改 hugo.toml：

baseURL = "https://example.com/"  
languageCode = "en-us"  
title = "My Notes"  
theme = "PaperMod"

重点是改theme

然后执行hugo就能转到public

hugo server -D

Reload Page

是tags的问题

我用obsidian，所以直接打开了content/post文件夹，但是里面还有一些我不想生成网页的文件，所以hugo生成网页的时候，对post内部的内容能否增加对文件夹的跳过？ 参考[https://discourse.gohugo.io/t/ignore-certain-files-and-directory-in-hugo/41960](https://blog.sangeeth.dev/notes/excluding-directories-from-build-in-hugo/)

在 hugo.toml / config.yaml 里加： Hugo config：

ignoreFiles:
  - "/content/post/private/.*"

整个文件夹完全不会参与构建

主题是否支持draft，支持的，但是我需要用吗？用吧，虽然刚开始可能不习惯

模板设定：在哪里，内容是什么

自动插入模板 创建文件的时候自动使用模板 不支持自动，必须用插件Templater Templater → Trigger Templater on new file creation（新文件创建时触发）

private文件夹迁移

obsidian的vault配置完成了，然后搞一下

  • github上传代码的时候不要把private、daily这些也上传：.gitignore
  • 上传一下最近的代码，别是网络问题
  • hugo生成网页的时候不要把这两个里面的也生成

为什么会有specialpage： hugo.yaml里面加上

params:
  disableSpecial1stPost: true

不要左右键，加上页码，而且一页显示的文章篇幅多一点 参考https://blog.csdn.net/gitblog_00200/article/details/151458895 打开分页模板文件layouts/_default/list.html，找到第102-104行和111-112行代码，确保ShowPageNums参数已启用。该参数控制是否在分页按钮中显示页码信息：

papermod的配置可以参考 https://github.com/wangyunzi/papermod/blob/master/config.yml
