图中判断成环

September 15, 2023 · map[avatar:images/avatar.jpg bio:Make milong famous again links:[map[name:GitHub url:https://github.com/milong26]] name:Milong]

求最小生成树的Kruskal算法里面遇到，这边写一下

无向图#

DFS#

对无向图有环=某个结点有一条边指向已访问过的结点，并且这个已访问过的结点不是上一步访问的结点 so问题可以转换成，如何区分已经访问过的节点，需要区分相邻的sibling、parent和间接的grandparent，当某节点有个sibling和某个grandparnt相同时，就是个环

修饰节点颜色#

我们不能仅仅使用一个 bool 数组来表示结点是否访问过。规定每个结点都拥有三种状态，白、灰、黑。开始时所有结点都是白色，当访问过某个结点后，该结点变为灰色，当该结点的所有邻接点都访问完，该节点变为黑色。

那么我们的算法可以表示为：如果在遍历的过程中，发现某个结点有一条边指向灰色节点，并且这个灰色结点不是上一步访问的结点，那么存在环。

为什么：假设灰色节点为G，它为灰色表示至少已访问过其中一个child节点，其中一个表示为parent，现在遍历到一个节点为grandchild，它可以指向G就表示它的sibling=grandparent，同时G不是grandchild上一步所访问的节点，so出现了环，这样正着推ok 反过来推，为什么需要3个状态，如果单纯两个状态就是下面的暴力递归。

再梳理一遍就能发现对DFS来说，就好比祖母和同学是同一个人这样

没有修饰，直接暴力递归#

节点就两个状态，访问过、没访问过。 我看不懂 个人理解： 更加纯正的DFS，不管三七二十一先遍历，遍历过了就给它标记成visited，然后一旦在遍历的时候遇到即将访问的point满足1. visited过了，2.不是直系parent，就直接输出是环，要等到所有遍历走过了才确认不是环。

BFS#

和dfs相比主要的思路是一样的 BUR，BFS需要增加一个parent数组（不是DFS里面用单个parent记录刚访问过的节点） BFS相比DFS用得不怎么多，有空再钻研

Union-Find#

也叫并查集。 对于无向图来说，在遍历边（u-v）时，如果结点 u 和结点 v 的parent相同，那么结点 u 和结点 v 在同一个环中。 图的全部顶点都各自属于一个集合；然后依次检查图中的所有边，如果边关联的两个顶点不属于同一个集合，就合并两个顶点所在的集合。若图中存在环，则一定会在检测到某条边的时候，发现它关联的两个顶点早已属于同一个集合。 并查集我不会啊

首先初始化所有元素的根为-1，-1代表根节点，接下来对于图中的每一条边(v1,v2)都并入集合，并入的方式为查找v1和v2的根节点，然后让v2的根节点作为v1的根节点，查找根节点的过程为：如果当前的结点根为-1，说明这个结点就是根，直接返回，否则再继续查找结点parent的根，直到找到root结点，这里因为只是判断环路，不需要压缩路径。

int findSet(int x){
    if(Parent[x] == -1){
        return x; // x is root
    }
    return findSet(Parent[x]);
}

并查集我真的看不懂 并查集find and union里面主要的是find吧，关键是怎么找root节点。

#include <bits/stdc++.h>
using namespace std;

//无向图的边
class Edge {  
public: 
	int src, dest;	
};
//边集数组
class EdgeSet {
private:
	int V, E;
	Edge *edge; 
	int size;   //边的数量 
public:
	EdgeSet(int V, int E);
	~EdgeSet() { delete [] edge; }
	void addEdge(int v, int w); //加入边到图中 
	int find(int parent[], int i);
	void merge(int parent[], int x, int y);
	bool isCycle();
};

EdgeSet::EdgeSet(int V, int E) {
	size = 0; 
	this->V = V;
	this->E = E;
	this->edge = new Edge[this->E];
}
void EdgeSet::addEdge(int v, int w) {
	edge[size].src = v;
	edge[size].dest = w; 
	++size;	
}

//使用并查集判断无向图中的环
//找到这个元素所在的子集
int EdgeSet::find(int parent[], int i) {
	if (parent[i] == -1) return i;
	return find(parent, parent[i]);
}
//进行两个子集的合并 
void EdgeSet::merge(int parent[], int xset, int yset) { 
	if (xset != yset) parent[xset] = yset;
}
//检查无向图中是否存在环 
bool EdgeSet::isCycle() {
	int parent[V];
	for (int i = 0; i < V; ++i) parent[i] = -1; 
	//迭代图中所有的边,找到每条边的两个顶点所在的子集
	//如果发现一条边,它的两个顶点属于相同子集,就存在环 
	//否则合并两个顶点的子集 
	for (int i = 0; i < E; ++i) {
		int x = find(parent, edge[i].src);
		int y = find(parent, edge[i].dest);
		if (x == y) return true;
		merge(parent, x, y);
	}
	return false;
}

int main() {
	/*
    0  
    | \  
    |   \  
    1----2 */
	EdgeSet es(3, 3);
	es.addEdge(0, 1);  //(0,1)
	es.addEdge(1, 2);  //(1,2)
	es.addEdge(0, 2);  //(0,2)
	es.isCycle() ? cout << "Contains Cycle!\n" : cout << "Contains No Cycle!\n"; 
	return 0;
}
