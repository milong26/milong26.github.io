树的计数相关问题

September 15, 2023 · map[avatar:images/avatar.jpg bio:Make milong famous again links:[map[name:GitHub url:https://github.com/milong26]] name:Milong]

光看名字就比较高级了:D 学习这个的动力来自于一个问题：对于包含n个顶点的无向完全图最多包含$n^{n-2}$棵生成树。

Prüfer序列#

Prüfer 序列可以将一个带标号 n 个结点的树用 [1,n] 中的 n-2 个整数表示。你也可以把它理解为完全图的生成树与数列之间的双射。常用于组合计数问题中。

对树建立序列#

对树建立Prüfer序列的步骤：每次选择一个编号最小的叶结点并删掉它，然后在序列中记录下它连接到的那个结点。重复 n-2 次后就只剩下两个结点，算法结束。 先删除节点再记录标号，重复n-2之后只剩下根节点没删。

堆构造#

听起来挺像建堆的过程，用堆的方法时间复杂度O(NlogN) 代码

// 结点是从 0 标号的
vector<vector<int>> adj;// adj的构造应该是：一行上每一个数表示和这行对应的点连接的节点序号
vector<int> pruefer_code() {
  int n = adj.size();
  set<int> leafs;
  vector<int> degree(n);
  vector<bool> killed(n, false);//killed表示已经删除的
  for (int i = 0; i < n; i++) {
    degree[i] = adj[i].size();//求每个点的度
    if (degree[i] == 1) leafs.insert(i);//如果这行只有自己一个节点，就是度为0（degree=1）=叶子节点
  }
  //求Prüfer code
  vector<int> code(n - 2);
  for (int i = 0; i < n - 2; i++) {
    int leaf = *leafs.begin();//叶子节点
    leafs.erase(leafs.begin());//每次都会删掉一个节点
    killed[leaf] = true;
    int v;
    for (int u : adj[leaf])//找leaf的邻接节点
      if (!killed[u]) v = u;
    code[i] = v;//记录leaf的邻接节点
    if (--degree[v] == 1) leafs.insert(v);//重新更新叶子节点
  }
  return code;
}

代码读懂了：一句话：按从小到大的顺序删除叶子节点，因为是树，code记录的就是被删除的叶子节点的parent 但是我不知道它有什么用.-.

线性构造#

在这个删除过程中，叶结点数是非严格单调递减的，删去一个叶结点，叶结点总数要么不变要么减 1。 维护一个指针 p。初始时 p 指向编号最小的叶结点。同时我们维护每个结点的度数，方便我们知道在删除结点的时侯是否产生新的叶结点。 步骤：

  1. 删除 p 指向的结点，并检查是否产生新的叶结点。
  2. 如果产生新的叶结点，假设编号为 x，我们比较 p,x 的大小关系。如果 x>p，那么不做其他操作；否则就立刻删除 x，然后检查删除 x 后是否产生新的叶结点，重复 2 步骤，直到未产生新节点或者新节点的编号 >p。
  3. 让指针 p 自增直到遇到一个未被删除叶结点为止； 新的问题：怎么检查是否产生新的叶节点？删除一个结点之后会产生度数影响的也只有parent节点，so检查parent是否有其它child即可 算法复杂度：发现每条边最多被访问一次（在删度数的时侯），而指针最多遍历每个结点一次，因此复杂度是 O(n) 的。 代码我真的看不懂QAQ，有源再看吧

性质#

在构造完 Prüfer 序列后原树中会剩下两个结点，其中一个一定是编号最大的点 n。 每个结点在序列中出现的次数是其度数减 1。（没有出现的就是叶结点）

反构建树#

根据性质可以重构，问题不大 so树->序列和序列->树均可行，Prüfer 序列与带标号无根树建立了双射关系。

Cayley 公式#

完全图的生成树#

完全图 $K_{n}$ 有 $n^{n-2}$ 棵生成树。

怎么证明？方法很多，但是用 Prüfer 序列证是很简单的。任意一个长度为 n-2 ,值域在 [1,n]内的整数序列都可以通过 Prüfer 序列双射对应一个生成树，算生成树个数问题就转变成求整数序列(数字可以重复)，用组合计数方法就是对每个位置都可以从n个数字里面随机抽一个，总共n-2个位置，得证

图连通方案数#

一个 n 个点 m 条边的带标号无向图有 k 个连通块。我们希望添加 k-1 条边使得整个图连通。求方案数。

证明：

假设现在有k个点，想要连通的方案数。 由上面prufer序列，一棵有k个节点的无根树，能构成$k^{k-2}$棵生成树。 设 $s_{i}$ 表示每个连通块的大小（点的数量）。

我们对 $k$ 个连通块构造 Prüfer 序列，然后你发现这并不是普通的 Prüfer 序列。因为每个连通块的连接方法很多。不能直接淦就设啊。于是设 $d_{i}$ 为第 $i$ 个连通块的度数。由于度数之和是边数的两倍，k个点有k-1条边于是 $\sum_{i=1}^{k} d_i=2k-2$ 。则对于给定的 $d$ 序列构造 Prüfer 序列的方案数是

$$ \binom{k-2}{d_1-1,d_2-1,\cdots,d_k-1}=\frac{(k-2)!}{(d_1-1)!(d_2-1)!\cdots(d_k-1)!} $$

这个圆括号是组合数学。为什么这里会有逗号，参考

对于第 $i$ 个连通块， 它的连接方式有 ${s_i}^{d_i}$ 种，因此对于给定 $d$ 序列使图连通的方案数是

$$ \binom{k-2}{d_1-1,d_2-1,\cdots,d_k-1}\cdot \prod_{i=1}^k{s_i}^{d_i} $$

现在我们要枚举 $d$ 序列，式子变成

$$ \sum_{d_i\ge 1，\sum_{i=1}^kd_i=2k-2}\binom{k-2}{d_1-1,d_2-1,\cdots,d_k-1}\cdot \prod_{i=1}^k{s_i}^{d_i} $$

好的这是一个非常不喜闻乐见的式子。但是别慌！我们有多元二项式定理：

$$ (x_1 + \dots + x_m)^p = \sum_{\substack{c_i \ge 0 ,\ \sum_{i=1}^m c_i = p}} \binom{p}{c_1, c_2, \cdots ,c_m}\cdot \prod_{i=1}^m{x_i}^{c_i} $$

那么我们对原式做一下换元，设 $e_i=d_i-1$，显然 $\sum_{i=1}^ke_i=k-2$ ，于是原式变成

$$ \sum_{e_i\ge 0，\sum_{i=1}^ke_i=k-2}\binom{k-2}{e_1,e_2,\cdots,e_k}\cdot \prod_{i=1}^k{s_i}^{e_i+1} $$

多项式定理： $$(x_1+x_2+…+x_n)^m=\sum_{\sum t=m}\binom{m}{t_1,t_2,…t_n}\prod x_i^{t_i}$$ 化简得到

$$ (s_1+s_2+\cdots+s_k)^{k-2}\cdot \prod_{i=1}^ks_i $$

即

$$ n^{k-2}\cdot\prod_{i=1}^ks_i $$
