1 Stable-Matching#

definition#

unstable pair：h-s：h更喜欢s，比h录取的其中一个s && s喜欢 h，比s分配到的h

ps我觉得stablematching的例子比↑要好

更新->给定一个perfect matching，对于h-s：如果（h心里，s比目前的好 && s心里h更好），h-s是unstable pair

stable assignment：没有unstable pair的方案

输入：两个集合H，S，有排序

Matching：h-s的集合。其中每个h至多在一对中，每个s至多在一对中

perfect matching：|M|=|H|=|S|=n（就是配对的个数跟h数、人数相等，并不一定人人满足）

stable matching：perfect matching+no unstable pairs

gale-shapley algorithm#

伪代码

初始化
while （存在h没有匹配 && 该h还没对每一个s发出申请）
{
s=[h的名单里的头一个&&h没发过申请]
if（s没配对） h-s配对加入M中
else if（s prefer h to current h')
h'-s 替换为h-s
else s拒绝h
}
return M

  • 至多n^2^次迭代while

  • gsa结果是一个匹配matching（至多出现一次）：不会同时同意两个的请求

  • gsa结果中，所有h都实现匹配：反证法

  • gsa结果中，所有s都实现匹配

  • gsa所得结果M中没有unstable pair：

    考虑任意配对h-s ，不在M中：

    1. h没有向s发出申请过： 因为h的申请是从好到坏的，所以h比起s，更偏好它目前的s'
    2. h向s发出过申请： s 拒绝过h s 比起h，更喜欢它目前的h'

​ 所以h-s不是unstable的

deterministic：确定的，相同输入，运行过程相同，结果相同

gsa得到的stable matching：过程不确定，但结果确定，只会有一个方案

hospital optimality#

s是a valid partner for h：存在stable M：h-s

hospital-optimal assignment：每个h都是最好的valid partner
